Burp Nuclei 插件编译与 SnakeYAML 依赖问题

发表于2025-03-10|更新于2026-03-10|安全研究
|浏览量:

一、背景说明

在使用 ProjectDiscovery / Nuclei 相关工具或插件时，发现一个常见问题：

  • Release 版本中仍使用旧版 SnakeYAML
  • 源码（pom.xml）中已经升级到了 SnakeYAML 2.x（如 2.5）
  • 导致：
    • 运行时仍存在安全漏洞（CVE）
    • 或在 Java 17+、严格 YAML 校验场景下出现兼容性问题

因此需要：

  • 自行基于最新源码打包 JAR
  • 并理解其 YAML 反序列化逻辑是否与 SnakeYAML 2.x 兼容

相关源码

  1. Nuclei Burp 插件
    源码位置：https://github.com/projectdiscovery/nuclei-burp-plugin

    从源码的 pom.xml 可以看到，依赖版本已经更新到 SnakeYAML 2.x，但是下载的 release 版本仍然报错，说明源码还没有完全更新到 release 中。

  2. Burp 官方插件
    https://github.com/PortSwigger/nuclei-template-generator

    这个插件的 pom.xml 连版本都没有更新。

二、自行编译解决方案

编译步骤

  1. 克隆最新源码
  2. 确认 pom.xml 中的 SnakeYAML 版本
  3. 执行编译命令：
mvn clean package -DskipTests

编译结果

编译成功后，在 target/ 目录下获取编译好的 JAR 文件：

target/
├── xxx.jar

注意事项

  • 编译前确保已安装 Maven 和合适的 JDK 版本（建议 Java 17+）
  • 如果遇到依赖问题，可以尝试清理 Maven 缓存：mvn dependency:purge-local-repository
  • 编译后的 JAR 文件可以直接在 BurpSuite 中作为插件加载

三、总结

对于安全工具插件的依赖问题，尤其是涉及 YAML 反序列化的场景：

  1. 不要完全信任 release 版本 - 有时源码已经修复但 release 未更新
  2. 学会自行编译 - 这是解决依赖问题的最可靠方法
  3. 关注 CVE 漏洞 - SnakeYAML 旧版本存在已知安全漏洞，及时升级很重要

本文记录了 Burp Nuclei 插件编译过程中遇到的依赖问题及解决方案，供后续参考。

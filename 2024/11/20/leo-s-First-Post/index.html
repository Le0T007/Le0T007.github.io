以下是通过 Hexo 配合 GitHub Pages 部署个人博客的详细步骤：

1. 安装必要工具

确保本地环境已准备好：

  1. 安装 Git

下载并安装 Git。

git –version
确保安装成功。
2. 安装 Node.js 和 npm
下载并安装 Node.js（会自动安装 npm）。
验证安装：
node -v
npm -v
3. 安装 Hexo
使用 npm 安装 Hexo：

npm install -g hexo-cli

2. 初始化 Hexo 博客项目

  1. 创建项目文件夹
    在终端中运行：

mkdir hexo-blog
cd hexo-blog
hexo init

  1. 安装依赖
    切换到项目目录后运行：

npm install

  1. 生成默认文件结构
    Hexo 会生成以下文件夹和文件：
    • _config.yml：站点配置文件。
    • source/：存放博客文章和页面。
    • themes/：存放主题。

3. 本地预览博客

  1. 生成静态文件
    在项目目录运行：

hexo generate
2. 启动本地服务器

hexo server

打开浏览器访问 http://localhost:4000 查看博客效果。

4. 配置部署到 GitHub Pages

  1. 创建 GitHub Pages 仓库

登录 GitHub，新建一个仓库，命名为 username.github.io（替换为你的 GitHub 用户名）。

  1. 安装部署插件

在 Hexo 项目中运行：

npm install hexo-deployer-git –save

  1. 配置 _config.yml 文件

编辑 Hexo 的主配置文件 _config.yml，找到 deploy 部分，修改为：

deploy:
  type: git
  repo: https://github.com/username/username.github.io.git
  branch: main

  1. 部署到 GitHub
    运行以下命令将博客部署到 GitHub Pages：

hexo deploy

如果提示 GitHub 用户名和密码，使用 个人访问令牌 (PAT) 替代密码。

5. 优化和自定义

  1. 更换主题

Hexo 默认主题为 landscape，可以从 Hexo Themes 下载更多主题。

• 下载主题：

git clone <theme-repo-url> themes/<theme-name>

• 修改 _config.yml 中的 theme 值为新的主题名称。

  1. 编写新文章

• 新建文章：

hexo new “My First Post”

• 编辑文章文件 source/_posts/My-First-Post.md。

  1. 部署更新

每次修改后运行：

hexo clean
hexo generate
hexo deploy
6. 绑定自定义域名（可选）

  1. 在 source/ 文件夹下创建 CNAME 文件：

echo “yourdomain.com” > source/CNAME

  1. 将域名解析到 GitHub Pages 提供的地址。

7. 查看博客

在浏览器中访问 https://username.github.io 查看已部署的博客。

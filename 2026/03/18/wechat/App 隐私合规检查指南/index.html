📱 App 隐私合规检查指南

基于个人信息保护法规的 App 合规检测实战指南

一、隐私政策同意前检测 🔒

核心原则

只要不点击”同意”按钮，就不能收集任何个人信息。

检查场景

  1. 点击”不同意”测试

    • 查看是否有”仅浏览模式”
    • 不同意后是否还能使用基本功能

  2. 隐私政策链接

    • 必须可点击阅读
    • 内容完整可访问

  3. 技术检测

    • 同意前不能有任何收集行为
    • 无论隐私政策是否写明

二、遍历 App 业务功能 📋

检查方法

  1. 尽可能遍历所有业务功能

    • 期间遇到权限申请要同意
    • 确保所有收集行为都被触发

  2. 权限申请合规性

    • 不能一揽子授权
    • 不提前索权
    • 用户主动触发具体功能再申请

  3. 收集频度检查

    • 不能超出业务功能实际需要
    • 仅关注特别场景

三、静默场景检测 🕐

前台静默

遍历后找个非业务功能页面进行静置

合规要求：

  • 若存在前台静默业务场景
  • 有合理业务目的
  • 隐私政策中必须明示”前台静默”也会收集

后台静默

遍历后将 App 置于手机后台，静置

合规要求：

  • 若存在后台业务场景
  • 有合理业务目的
  • 隐私政策中必须明示”后台期间”也会收集

四、个人信息收集方式 🔍

直接收集

用户可感知 — 需要用户主动填写或授权

  • 填写手机号
  • 填写身份证号
  • 授权相机权限
  • 授权位置权限

间接收集

用户无法感知 — 读取设备信息、调用敏感权限

  • Android ID
  • IP 地址
  • MAC 地址
  • 调用 READ_PHONE_STATE 等权限

第三方共享

App 从其他方（公司或平台）共享获得：

隐私政策需要明示共享或获得的信息

五、合规检查场景分类 ✅

1️⃣ 纯隐私政策内容完整

隐私政策逐一列出所有收集行为

2️⃣ 纯查 App 功能页面

同步告知内容的完整性

要求：

  • App 要在显著位置公示客服热线
  • App 要同步告知用户目的

3️⃣ 纯技术检测

同意隐私政策前，不管隐私政策写没写，就是不能收集

4️⃣ 技术检测 + 隐私政策 / 功能页面 + 隐私政策

  • 未逐一列出
  • 申请权限目的需参考隐私政策

六、敏感权限申请合规性 ⚠️

1. 权限声明限制

App 仅能声明业务功能相关权限

检测点： 配置文件中不能过度声明权限

2. 申请时机

  • 不一揽子授权
  • 不提前索权
  • 用户主动触发具体功能再申请所需要的权限

3. 申请时同步告知

  • 同步告知用户目的
  • 目的描述明确、具体
  • 与当前功能一致

4. 用户拒绝后的处理

必须遵守：

  1. App 不能自动退出或关闭
  2. 非必要权限不能频繁索权（48h 内再次索权）
  3. 必要权限：用户主动触发后可以弹窗
  4. 拒绝非必要权限，不能影响其他业务功能使用

5. 替代方案

宜提供不需要权限的替代解决方案

七、检测工具和方法 🛠️

AppScan 工具

  • 自动化检测个人信息收集行为
  • 检测权限调用情况
  • 生成合规报告

技术检测场景

检测项 方法
设备信息读取 抓包分析
权限调用 日志监控
网络传输 流量分析
SDK 行为 静态分析

八、常见问题汇总 ❗

违规收集场景

  1. 未同意隐私政策前收集
  2. 隐私政策未逐一列出收集行为
  3. 收集频度超出业务需要
  4. 权限过度声明
  5. 频繁索权（48h 内）
  6. 拒绝权限后 App 退出
  7. 未明示静默收集场景

合规整改建议

  1. 完善隐私政策内容
  2. 调整权限申请时机
  3. 提供替代解决方案
  4. 明示特殊收集场景
  5. 建立合规检测流程

九、法规依据 📚

  • 《个人信息保护法》
  • 《网络安全法》
  • 《数据安全法》
  • GB/T 35273-2020《信息安全技术 个人信息安全规范》
  • 《App 违法违规收集使用个人信息行为认定方法》

十、检查清单 ✅

发布前必查

  • 隐私政策完整且可访问
  • 同意前无任何收集行为
  • 权限申请时机合规
  • 权限目的明确具体
  • 拒绝权限后不影响基本功能
  • 明示静默收集场景
  • 公示客服热线
  • 提供替代解决方案
